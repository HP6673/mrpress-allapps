## 📖 About the Project

**All Apps by MrPress** was born out of a simple frustration: too many tools scattered across too many tabs, bookmarks, and websites. The goal was straightforward — build one website that acts as a personal launchpad for everything.

This project is more than just a links page. It's a fully integrated, all-in-one web platform where each app or tool is embedded, linked, or natively built directly into the site. Whether you're looking for productivity utilities, fun tools, or handy web apps, **All Apps** has you covered.

Hosted entirely on **GitHub Pages**, the project is version-controlled, publicly accessible, and continuously evolving. MrPress maintains and updates the platform regularly, adding new apps and refining the experience over time.

### 🎯 Goals

- **Consolidation** — Bring everything into one easy-to-navigate hub
- **Speed** — Keep the platform lightweight and fast for all users
- **Accessibility** — Make it usable across all devices and screen sizes
- **Openness** — Keep the codebase open so others can learn and contribute
- **Consistency** — Provide a unified, polished UI across all apps and sections

## 🚀 Getting Started

Follow these steps to get a local copy of the project up and running on your machine.

### Prerequisites

Before you begin, make sure you have the following installed:

- A modern web browser (Chrome, Firefox, Edge, Safari)
- [Git](https://git-scm.com/) — for cloning the repository
- *(Optional)* [Node.js](https://nodejs.org/) — if you want to use a local development server

You can verify Git is installed by running:

```bash
git --version
```

### Installation

1. **Fork the repository** *(optional but recommended if you plan to contribute)*

   Click the **Fork** button at the top right of this page.

2. **Clone the repository** to your local machine:

   ```bash
   git clone https://github.com/MrPress/all-apps.git
   ```

3. **Navigate into the project directory:**

   ```bash
   cd all-apps
   ```

### Running Locally

#### Option 1 — Open Directly in Browser

If the project is purely static (HTML/CSS/JS), simply open the entry point:

```bash
open index.html
# or on Windows:
start index.html
```

#### Option 2 — Use a Local Dev Server (Recommended)

For the best local development experience, use a simple HTTP server:

```bash
# Using Node.js (npx)
npx serve .

# Or using Python 3
python -m http.server 8080
```

Then open your browser and visit:

```
http://localhost:8080
```

#### Option 3 — VS Code Live Server

If you use **Visual Studio Code**, install the [Live Server extension](https://marketplace.visualstudio.com/items?itemName=ritwickdey.LiveServer), right-click `index.html`, and select **"Open with Live Server"**.

## 🤝 Contributing

Contributions are what make open source such a fantastic ecosystem to be a part of. Any contributions you make to **All Apps by MrPress** are **greatly appreciated**.

### How to Contribute

1. **Fork** the repository
2. **Create** your feature branch:
   ```bash
   git checkout -b feature/AmazingFeature
   ```
3. **Commit** your changes with a clear message:
   ```bash
   git commit -m 'Add: AmazingFeature that does X'
   ```
4. **Push** to your branch:
   ```bash
   git push origin feature/AmazingFeature
   ```
5. **Open a Pull Request** and describe what you've changed and why

### Code Style

To keep the codebase clean and consistent, please follow these guidelines:

- Use **2 spaces** for indentation (no tabs)
- Write **semantic HTML** — use the right element for the right job
- Keep **CSS organized** — group related rules, use comments for sections
- Write **readable JavaScript** — clear variable names, comments where needed
- Test in **multiple browsers** before submitting a PR

### Reporting Bugs

Found a bug? Please [open an issue](https://github.com/MrPress/all-apps/issues/new) and include:

- A clear title and description
- Steps to reproduce the bug
- Expected vs. actual behavior
- Screenshots if applicable
- Browser and OS details
